# 대화 영속성(메모리) 조사 — Mem0 vs Letta (2026-06)

> Molly는 "사용자를 기억하는 친구"가 페르소나의 본질. 대화 영속성을 위한 메모리 아키텍처 조사.
> 분담: **우리 = Mem0**(메모리 레이어), 다른 팀원 = **Letta**(스테이트풀 에이전트 런타임).

## 1. 메모리 종류 (companion 관점)
| 계층 | 내용 | 예 |
|---|---|---|
| 작업기억(세션) | 현재 대화의 최근 턴 | "방금 저녁 얘기함" |
| 프로필/사실(영속) | 사용자에 대한 지속 사실 | 이름·직업·취향·가족·목표 |
| 에피소드 요약(영속) | 지난 대화 요약 | "지난주 회사 힘들다고 함" |

Molly는 친구라 transcript RAG보다 **사실/프로필 메모리**가 핵심.

## 2. Mem0 vs Letta — 근본 차이
| | **Mem0 (메모리 레이어)** | **Letta/MemGPT (에이전트 런타임)** |
|---|---|---|
| 철학 | 메모리만 서비스로 붙임 | 에이전트가 Letta 런타임 "안에서" 돎 |
| 누가 관리 | 프레임워크가 백그라운드 추출(개발자가 add/search) | 에이전트(LLM)가 도구콜로 자기 메모리 편집 |
| 메모리 모델 | 벡터스토어 + (옵션)그래프 | OS식 3계층 Core(RAM)·Recall(디스크)·Archival(콜드) |
| 내 코드 | 에이전트 루프·도구 내가 소유 | 루프·도구·상태·메모리 Letta가 소유 |
| Lock-in | 최소(API 경계 깔끔) | 큼(인프라 전체가 Letta) |
| 적합 | "사용자 기억" 기능이 목적인 소비자 앱 | 장기 자율 에이전트 |

→ **Molly엔 Mem0**: 파이프라인(STT→moly-server→TTS)이 이미 있고 "기억"만 추가하면 됨. Letta는 런타임 전체를 가져와야 해 우리 구조와 충돌.

## 3. Mem0 내부 동작 — 2단계 파이프라인
**① 추출**: LLM이 `최근 메시지 + 대화 요약`에서 원자적 사실 추출, 잡담 제거.
**② 갱신**: 후보 사실을 벡터검색으로 유사 기존 기억 top-K 조회 후 LLM이 결정:
- **ADD** 새 기억 / **UPDATE** 보완 / **DELETE** 모순 제거 / **NOOP** 변경 불필요

→ 단순 RAG와 다른 핵심: 쌓기만 하는 게 아니라 **모순·변화를 능동 관리**("예전엔 면 싫다더니").

저장: 임베딩 + 메타(user_id, timestamp). 옵션 Mem0g(그래프): 엔티티=노드, 관계=엣지.
조회: 쿼리 임베딩 → 유사도 검색 → 관련 기억 → 프롬프트 주입.

## 4. 운영 동작
- **쓰기(add)**: async 백그라운드 — 추출 0.5~2초 걸려도 응답 안 막음
- **읽기(search)**: managed 100~400ms — 턴마다 X, 세션시작/필요시
- 추출용 LLM은 저렴+적당한 걸로(대화 LLM과 분리)

## 5. 배포: Managed vs OSS Self-host
| | Managed | Self-host |
|---|---|---|
| 시작 | 5분, API키 | Docker 3개(FastAPI+pgvector+Neo4j) |
| 데이터 주권 | Mem0 서버 | 내 인프라(프라이버시 제어) |
| 주의 | 외부 저장 | 인증·로그인 없음, history SQLite |

→ **현재: managed로 빠르게 검증**. 프로덕션/프라이버시 단계에서 self-host 재검토.

## 6. 영속성 개발 시 고려사항 (체크리스트)
1. **지연** — 읽기 세션시작/캐시, 쓰기 async (실시간 음성 0.93초 유지)
2. **추출 정책** — 원자적 사실만, 과추출 금지
3. **모순/갱신** — UPDATE/DELETE로 변화 처리
4. **격리** — user_id로 사용자별 분리, 기억 누수 금지
5. **프라이버시/제어** — 사용자가 기억을 보고·수정·삭제(개인정보보호법, 신뢰)
6. **비용** — 추출 LLM 토큰 비용, 저렴 모델
7. **품질 평가** — 틀린 기억이 관계를 망침, 테스트셋 필요
8. **주입 예산** — top-K 제한, 관련도순
9. **저장 인프라** — managed면 불필요, self-host면 벡터DB
10. **페일세이프** — 메모리 죽어도 대화는 돌아가야(enhancement, not blocker)
11. **콜드스타트** — 신규 사용자 자연스럽게
12. **세션 경계** — 언제 add? 세션 요약 전략
